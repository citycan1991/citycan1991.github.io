机器学习入门——基本概念

  • 机器学习

最近在看一些机器学习相关的资料，发现机器学习这门学科对于入门者来说其实并不友好。首先，机器学习作为一门横跨计算机学、统计学、概率论等的多领域交叉学科，不仅要理解很多概念和专业名词，还需要非常扎实的数学基础，其次还需要很多计算机中的算法基础。

所以，我们先从最基本的概念谈起。

什么是机器学习？

回答这个问题之前，我们可以先回忆一下，人类是怎么学习的。人类除了从书籍、报纸、网络等媒介直接获取知识外，还有一种通过不断的自我总结来学习的方式。对于很多我们无法从搜索引擎中搜索到标准答案的问题，往往后一种方式更加有效。比如，你有一个朋友A，当我问你A这个月会不会请你吃饭的时候，你会在大脑里面回忆，A以前请过我吃饭吗，上次请我吃饭是什么时候等等一系列问题。最终，你会根据以往的经历总结出经验，如果A长期以往的坚持每个月请你吃饭，那他这个月也一定会请你吃饭。如果A从来没有请你吃过饭，那可能你这个月需要找你另外的朋友蹭饭了。

“A computer program is said to learn from experience E with respect to some class of tasks T and performance measure P, if its performance at tasks in T, as measured by P, improves with experience E”

以上是引用维基百科中对机器学习的定义。简单的来说，与人类通过各种各样过往的经历总结出经验来帮助自己对未来进行分析一样，机器学习就是计算机利用已有的数据，得出了某种模型，并利用此模型预测未知数据的一种方法。

机器学习与人工智能

可能你经常会有这样的感受，为什么说到人工智能总会扯到机器学习，谈起机器学习也经常扯上人工智能。二者有何关联呢？要让计算机像“人”一样，我们先回想一下人类有具有的一些能力：记忆力、计算能力、搜索能力能力、学习能力等等。在科技发达的今天，计算机拥有强大的数据存储能力，有云计算提供的高性能计算能力、有搜索引擎提供的快速准确的搜索能力，而机器学习就是计算机迈向“人工智能”最为关键的一步，计算机的自学习能力。如果说“人工智能”是目标的话，机器学习就是方法，让计算机变得“智慧”的方法。

机器学习与深度学习

在没有接触机器学习之前，我也纳闷过，到底机器学习是啥，深度学习又是啥，我应该学机器学习还是深度学习。后来经过学习发现，其实深度学习只是机器学习中的一个方向。深度学习源自于很早之前就有的神经网络算法，本来一直很低调，因为机器学习的出现变得大红大紫。简单的讲，神经网络是指模拟神经元分层处理的一种算法，传统的神经网络算法在多隐层场景下其性能堪忧，但机器学习的出现，改善了多隐层的问题，从而产生了深度神经网络，即我们说的深度学习。

机器学习的分类


按照学习方法的不同，可以粗略的划分为两类：

  • 监督学习：可以简单的理解为通过对已知数据进行模型训练，然后来预测未知数据的学习方法。这类学习方法主要用于处理分类、回归等问题。
  • 无监督学习：直接通过现有的数据进行建模，挖掘其内在的联系，注重的是对已知数据的分析。无监督学习常常用于聚类、降维等。

上面提到了几个概念：分类、回归、聚类、降维。那么这几个概念到底是什么意思呢？

  • 分类：这里分类的含义与字面意思差不多，主要是指机器学习中对离散型数据的一种处理方法，输出主常常为是/否这样的定性值。例如，我知道了世界上所有已知动物的外观、习性、品种等众多关键性的特征，那么对于一个新的未知物种，就可以通过这些特征数据得出它的科属。常用的分类器有朴素贝叶斯、决策树、kNN等。
  • 回归：和上面分类相对，回归主要是处理连续性数据，输出的结果一般为数值，而不是分类中的阿猫、阿狗这样的类型值。例如根据某地区历年的降雨数据，通过回归模型，预测出今年的该地区的降雨量。其实分类中的一些算法模型也可以用于回归中。
  • 聚类：俗话说“物以类聚，人以群分”。那么聚类就是对海量的数据进行分类。聚类属于无监督学习的范畴，即不需要训练数据，通过特征值直接将已知数据切割为几类，或者给出每条数据属于某个类的概率。K-means就是典型的聚类算法。聚类也常常用于商品推荐等场景。
  • 降维：同样属于无监督学习的还有降维。我们知道，数据的维度越多，用于机器学习的样本量也就越大。那么降维就是去除无用噪声，进行特征选择或者特征提取。降维可以通过主要成分分析(PCA)等算法实现。

机器学习处理数据的简单步骤

实际项目中可能机器学习是一个非常浩大的流水线工作，需要非常多的步骤，这里简单的介绍一下最基本的流程：

  • 数据预处理：预处理的方式方法多种多样，具体需要哪些步骤要看实际的数据。常用的步骤有：数据清洗/数据变换/噪声数据处理/特征降维等。

  • 模型选择：根据现有数据及处理目的，选择合适的模型，通常模型选择不是唯一的，可以多选择几个模型进行数据训练

  • 数据拆分：将现有数据按照一定比例（例如80:20/75:25）拆分为训练数据和测试数据。当然，这个步骤不是必须的，如果你有另外的测试数据的话。

  • 数据训练：用现有的数据，对模型进行训练。
  • 模型优化：这个步骤应该是最痛苦的，对当前的模型进行调参啊各种玄学操作，提高模型的性能、准确率等

机器学习的一些学习感受

其实我自己也是初学者，谈不上总结什么经验教训。初识机器学习，兴趣盎然，但是一段时间之后，就充斥着复杂的数学公式、深奥的算法。所以，我最直观的感受就是，要想清楚自己的学习目的。

  • 如果你是和我一样，对新技术感兴趣，想看一看，学一学，那你可以走马观花，不必拘泥于一个数学公式的推导或者一个算法解释，跳出来从整体看全貌。你会发现，哇，机器学习还是挺有意思的，甚至对我们的未来产生了美好的憧憬。
  • 如果是应用为目的的话，其实现在各种机器学习的包啊库啊已经非常完善了，那么需要具备理论基础的前提下快速上手，提高对各种库的熟练度。我认为应用最难的地方在于对模型的选择（毕竟没有最好的模型，只有最合适的模型），还有对模型的优化（各种玄学调参23333）。这两个点，都是直接影响最后数据的准确性。
  • 如果你是以研究为目的的话，对不起，我帮不上你。菜鸟并没有办法为大神指路。
